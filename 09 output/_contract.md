# Output capture — the contract

One folder per concept run: `09 output/[concept]/`. Inside it, one subfolder per derivation: `[concept]/[tool]-[n]/`.

## Per derivation, at the moment it's made

Not at the end of the session. The moment a variation exists, before making the next one:

1. **The artifact** — screenshot or export of the result, plus the code where the tool gives it (v0 and Paper do).
2. **`capture.md`** — small, four things:
   - **Tool:** Paper / v0 / Figma Make, and the date
   - **Prompt:** which compiled prompt from `08 briefs/[concept]/prompts/` ran (link or filename), plus any mid-run steering that changed it
   - **Expressions in play:** which expression folders this derivation drew on
   - **Commentary:** the honest read, raw, transcribed if spoken. Not "looks good" — the round-one kind: what's working and why, what clashes and whether the clash is good tension, what this rules out, what surfaced that wasn't asked for, what needs to get made before this can be honestly judged.

3. **Copy back** — the artifact + capture.md get copied into each source expression's `03 expressions/[name]/explorations/`. The expression folder is where the next round reads; this folder is the run record. If the duplication bloats at volume, `explorations/` becomes a pointer back to the 09 record instead of a second copy.

## Why this exists

Round one produced volume across three tools and the judgment lived in Jack's head and scattered chats. The ideas-and-thinking that come up while producing — "I like this script because it clashes with X and that's good tension" — are what concepts get pulled out of later. Captured in the moment they compound; captured at the end of the day they're gone.

Run-level docs (like Rd 1's `run-1/01-concept.md` — the locked concept, anchor devices, tone target) still belong at the `[concept]/` level when a run locks something. The per-derivation capture is what round one lacked and this round adds.
