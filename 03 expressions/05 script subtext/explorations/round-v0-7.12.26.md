# Round — v0 — 7.12.26

## What was tried

Four live variations of the script-subtext move — a serif small-caps label ("THE ROOMS") with a script phrase beneath it, sitting over a two-room excerpt of the catalog grid. Routes:

- `/e05/var-1` — script face A (font-script-a) at modest subtext scale. The default read: label leads, script whispers.
- `/e05/var-2` — script face C (font-script-c), same scale. A drier, more upright script against the same chrome.
- `/e05/var-3` — script face B at slightly larger scale — the script starts asking for attention.
- `/e05/var-4` — script face A blown up past the label (the "subtext becomes the headline" failure-test). Deliberately overcooked to find the ceiling.

All variations use real catalog data from `lib/data.ts` (Queen Historic, Rock Hudson Suite) and current CDN photography with honest captions — no finished-state claims.

## What was learned

- The move works when the script stays subordinate: var-1 and var-2 read as a hand annotating a formal document, which is the Paisano register.
- Var-4 confirms the ceiling: once the script outweighs the small-caps label, the page reads wedding-invitation, not West Texas hotel. The script phrase also gets harder to read at width — "forty-two rooms, nine ways to stay" wants to be caught in one glance.
- Script face choice matters less than scale. All three faces survive at subtext size; none survive at headline size.
- The em-dash data line under each room card (from 16's receipt register) coexists fine with the script subtext — they occupy different registers and don't fight.
